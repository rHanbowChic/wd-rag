<h1 id="toc0"><span>Summary</span></h1>
<ul>
<li>a template is a special page that defines common items to display on all pages in a category</li>
<li>templates are easy way to change the layout for all pages in a category in one place</li>
<li>the name of the special page defining the layout for a category is <em>category</em><tt>:_template</tt> (examples below)</li>
</ul>
<h1 id="toc1"><span>Details</span></h1>
<p>To create a template for a category you create a page called "_template" in that category. For the default category, the template page is called "_template". For example:</p>
<div class="code">
<pre>
<code>_template
category:_template
bugs:_template
proposals:_template</code>
</pre></div>
<p>Pages that start with an underscore are not listed by other modules and hidden in most cases.</p>
<h2 id="toc2"><span>Create your live template page</span></h2>
<p>Create a page called _template but do not leave it empty: as a minimum you need to add <span style="white-space: pre-wrap;">%%content%%</span>. This tells the template to simply display the current content of the page in the category.</p>
<div class="wiki-note">
<p>You cannot use the NewPage module to create a hidden page (a page whose name starts with an underscore — "_"). You must append <em>category:_template</em> to the URL in the address bar and hit enter to create live template pages. On the feedback site, there is a wish to change it this behavior. If you also feel this way, <a href="http://feedback.wikidot.com/wish:404">rate it up</a>.</p>
</div>
<br/>
The purpose of a template is to define a layout of pages within a category. Here is a simple template that forces the height of the content box.
<p>Here is a typical template,which adds a comments module below the page content:</p>
<div class="code">
<pre>
<code>+ Original content of the page below

%%content%%

----------------

[[module Comments]]</code>
</pre></div>
<p>It is easy to imagine what it really does. When this markup is saved as <tt>some-category:_template</tt>, all pages from the <tt>some-category</tt> will be combined with the template during rendering. <strong>Content of the page will be substituted into the %%content%% tag</strong>. This way we can add headers, side bars, navigation elements, modules and comment boxes to the template and the viewed page will automatically have it.</p>
<p>When you try editing a page from the category, only the "inside content" is editable. Template is applied only when viewing the compiled page.</p>
<h2 id="toc3"><span>Splitting the content</span></h2>
<p>Let us start with an example of a template:</p>
<div class="code">
<pre>
<code>[[div style="float:right; width: 200px; border: 1px solid #999; margin: 10px;"]]
%%content{1}%%
[[/div]]

%%content{2}%%

[[table]]
[[row]]
[[column]]
%%content{3}%%
[[/column]]
[[column]]
%%content{4}%%
[[/column]]
[[/row]]
[[/table]]

==== 

This will be content of the side bar.

==== 

The main content.

==== 

This will go into the left cell of the table.

==== 

And this into the right cell.</code>
</pre></div>
<p>There are two things described above: splitting content and default page content.</p>
<p><strong>Splitting</strong> allows you to create sections in the page and manipulate them separately in the template. Sections are separated by a series (4 or more) "equals" characters and are referred to as %%content{X}%% in the template. You can access them in any order within the template.</p>
<p>Using this you can easily create advanced layouts (like multicolumn, multi-navigation etc.) and make editing and maintenance much easier.</p>
<h2 id="toc4"><span>Default content</span></h2>
<p>Also in the template you can provide the <strong>default initial content for newly created pages</strong>. Such content is separated from the template by the '====' tag - the same as for splitting the content. To make it visually different you can make it longer — e.g. 10 characters. When a user wants to start a new page in the category, this content will be placed in the editor.</p>
<p>Also, when default content is defined for the template, there will be no option for selecting templates from the <tt>templates</tt> category (our previous templating mechanism).</p>
<h2 id="toc5"><span>Escaping the ==== tag</span></h2>
<p>Since you might want to use the "====" tag in the content of a page and NOT as a splitter, there is a way to escape the splitter and prevent the default action. Surround the splitter with "@@" like this:</p>
<div class="code">
<pre>
<code>@@====@@</code>
</pre></div>
<p>You can also escape the "====" tag by adding a space. This is the only way to escape it inside a code block. <em>Note: if you copy/paste code that contains '====', make sure it does not still have a space at the end of the line, or it will not work!</em></p>

<p>Date formatting:</p>
<ul>
<li>All _at fields are dates and allow a custom format via the <tt>|<em>format</em></tt> specifier.</li>
</ul>
<p>Most tokens from PHP's <a href="http://php.net/manual/en/function.strftime.php">strftime</a> are accepted. You may find <a href="http://community.wikidot.com/howto:frontforum-date-variable">the howto</a> contributed by community useful.</p>
<p>An example of how these can be used in a blog could be (e.g. in <tt>blog:_template</tt>:</p>
<div class="code">
<pre>
<code>by %%created_by%% on %%created_at|%e %B %Y%%
rating: %%rating%%, tags: %%tags%%

%%content%%</code>
</pre></div>
<h2 id="toc7"><span>Changing the template</span></h2>
<p>When you edit the <tt>_template</tt> page for the given category, all pages from the category will be recompiled to include the changed template.</p>
<h2 id="toc8"><span>Hidden pages</span></h2>
<p>Pages whose name starts with an underscore (like "_start", "_header", etc.) are 'hidden' pages and are not affected by the template.</p>
<h2 id="toc9"><span>Inexisting page templates</span></h2>
<p>If you will go to the address of the page that not exists, you'll a get a note similar to the following:</p>
<div style=" background-color: #38D; -webkit-border-radius: 5px; -moz-border-radius: 5px; border-radius: 5px; border: 1px solid #38D; margin: 0 2em; padding: 0 1em;">
<h1 id="toc10"><span>The page does not (yet) exist.</span></h1>
<p>The page <em>james</em> you want to access does not exist.</p>
<ul>
<li><a class="wiki-standalone-button" href="javascript:;" onclick="WIKIDOT.page.listeners.editClick(event)">create page</a></li>
</ul>
</div>
<p>You can use a template which will change the default "non-existing page" message to the custom one. All you need to do is to edit a page:</p>
<ul>
<li><a href="http://site-name.wikidot.com/_404">http://site-name.wikidot.com/_404</a> — it will change the message for all non-existing pages on your site</li>
<li><a href="http://site-name.wikidot.com/category:_404">http://site-name.wikidot.com/category:_404</a> — it will change the message for all non-existing pages within particular category</li>
</ul>
<p>Note: Of course, you need to change "site-name" to your Site's address and "category" to your own category.</p>
<p>In the template, you may want to have a link which allow users to create a page which don't exist. To do this, simply use:</p>
<div class="code">
<pre>
<code>[[button edit]]</code>
</pre></div>
<p>You may also want to change the title of _404 page to desired one, e.g. "This site does not exist".</p>
<p><span style="white-space: pre-wrap;">%%404_page_name%%</span> variable puts a name of the page which doesn't exist yet, i.e. if your template looks like this:</p>
<div class="code">
<pre>
<code>+ Oops! We can't find the page: //@@%%404_page_name%%@@//

Click [[button edit text="here"]] to create one.</code>
</pre></div>
<p>And you will go to the page <a href="http://site-name.wikidot.com/a-great-new-page">http://site-name.wikidot.com/a-great-new-page</a> it will result in:</p>
<div style=" background-color: #38D; -webkit-border-radius: 5px; -moz-border-radius: 5px; border-radius: 5px; border: 1px solid #38D; margin: 0 2em; padding: 0 1em;">
<h1 id="toc11"><span>Oops! We can't find the page: <em>a great new page</em></span></h1>
<p>Click <a class="wiki-standalone-button" href="javascript:;" onclick="WIKIDOT.page.listeners.editClick(event)">here</a> to create one.</p>
