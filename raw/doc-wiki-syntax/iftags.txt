
<div class="page-title">
                    Iftags
                </div>
<div class="breadcrumb">
<a href="/doc:start">Documentation</a> » <a href="/doc-wiki-syntax:start">Wiki Syntax</a> » Iftags
                    </div>
<div id="page-content">
<p>Tags are kind of special labels for a page, manually added in by editors by clicking on the <strong>tags</strong> link at the page options buttons on bottom of a page. Every tag can be max 64 characters long, tags are "space" separated and there is no limit of tags per page. Tags are very useful to label pages and then it's easy to create Tag Cloud, which allow to find interesting pages or topics much faster.</p>
<p>Special tags start with an underline: they are not automatically shown in tag clouds, but they can be used as special limitations in [[iftag]] conditions. Tags can be used in ListPages Module with generic conditions ( +, - ) too.</p>
<p><strong>Iftag</strong> is a special condition question. You can use it on every page to "react" on tags and set up on the particular page used .</p>
<p>Syntax:</p>
<div class="code">
<pre>
<code>[[iftags +tag1 -tag2 tag3]] ... [[/iftags]]</code>
</pre></div>
<br/>
where the +/-"tag#" stands for the requested tag-indexes.
<ul>
<li>+ before a tagname means - this tag must exist, (tag without a modifier works in a same way)</li>
<li>- before a tagname means - this tag must not exist.</li>
</ul>
<p>Example:</p>
<div class="code">
<pre>
<code>[[iftags +science]]
This page is labeled as: science.

Click here to view more science articles &gt;
[[/iftags]]

[[iftags +bug -fixed]]
This is a bug, but it's not fixed yet.
[[/iftags]]</code>
</pre></div>
