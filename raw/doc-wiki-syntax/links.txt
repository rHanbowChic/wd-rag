<h2 id="toc0"><span><a name="internal"></a>Internal links</span></h2>
<p>Unlike some other wiki engines Wikidot.com does not process SquashedAndCapitalized words as page links. Instead any link should be marked with 3 nesting square brackets.</p>
<p>If a page address contains disallowed characters the address will be "unixified" to contain only allowed chars. The displayed name however will maintain original form.</p>

<td>using raw page name</td>
</tr>

</tr>

<p>Adding underscore to <strong>a</strong> element <strong><span style="white-space: pre-wrap;">[[a_ ]]</span></strong> will truncate whitespaces around it which prevents creation of random <a href="/doc-wiki-syntax:paragraphs-and-newline">new lines and paragraphs</a>. It's simplifices creation of complex HTML syntax like <a href="http://getbootstrap.com/components/">Bootstrap components</a></p>
<h2 id="toc2"><span><a name="anchors"></a>Anchors</span></h2>
<p>To place an anchor use <tt><span style="white-space: pre-wrap;">[[# anchor-name]]</span></tt> syntax. To refer to an anchor (and scroll to it) use <tt><span style="white-space: pre-wrap;">[#anchor-name text to display]</span></tt>.</p>
<h2 id="toc3"><span><a name="emails"></a>Emails</span></h2>
<table class="wiki-content-table">
<tr>

<tr>
<td><tt><span style="white-space: pre-wrap;">support@example.com</span></tt></td>
<td><span class="wiki-email">moc.elpmaxe|troppus#moc.elpmaxe|troppus</span></td>
<td>simple inline email</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[support@example.com email me!]</span></tt></td>
<td><span class="wiki-email">moc.elpmaxe|troppus#!em liame</span></td>
<td>custom anchor</td>
</tr>
</table>
<p>Although we discourage anyone from putting his/her email address on the web, Wikidot engine provides a simple scrambling mechanism to prevent automated bots from reading emails. Each email is scrambled and it is decoded in the client's browser. So it is not 100% spam-safe, but much safer than plain-text emails.</p>
<h2 id="toc4"><span><a name="interwiki"></a> InterWiki</span></h2>
<p>To link directly to a Wikipedia article you can use a syntax:</p>
<table class="wiki-content-table">
<tr>
<th>what you type</th>
<th>what you get</th>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[wikipedia:Albert_Einstein]</span></tt></td>
<td><a href="http://en.wikipedia.org/wiki/Albert_Einstein" onclick="window.open(this.href, '_blank'); return false;">Albert_Einstein</a></td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[wikipedia:Albert_Einstein Albert]</span></tt></td>
<td><a href="http://en.wikipedia.org/wiki/Albert_Einstein" onclick="window.open(this.href, '_blank'); return false;">Albert</a></td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[wikipedia:it:Albert_Einstein Albert]</span></tt></td>
<td><a href="http://en.wikipedia.org/wiki/it:Albert_Einstein" onclick="window.open(this.href, '_blank'); return false;">Albert</a></td>
</tr>
</table>
<p>Other links defined by example:</p>
<ul>
<li><tt><span style="white-space: pre-wrap;">[google:free+wiki]</span></tt> - search google for the "free wiki" term</li>
<li><tt><span style="white-space: pre-wrap;">[dictionary:wiki]</span></tt> - look up definitions of the word <em>wiki</em> from dictionary.reference.com</li>
</ul>
<h2 id="toc5"><span><a name="magicuris"></a> Magic URIs</span></h2>
<p>Magic URIs (or Magic Links) are the way to control pages within the URL address.</p>
<table class="wiki-content-table">
<tr>
<th>what you type</th>
<th>what you get</th>
<th>comments</th>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name#_editpage Edit]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name#_editpage">Edit</a></td>
<td>Goes to the page with the edit mode already opened</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name/title/whatever Edit with title]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name/title/whatever">Edit with title set</a></td>
<td>Works only with not existing pages. When you go to edit mode on such page, the title will be set to 'whatever'. May be combined with /edit/true, parentPage/page-name etc.</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name/parentPage/parent-page-name Edit with parent page set]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name/parentPage/parent-page-name">Edit with parent page set</a></td>
<td>Works only with not existing pages. When you go to edit mode on such page, the parent page will be set to 'page-name'. May be combined with /edit/true, title/whatever etc.</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name/noredirect/true Page without redirect]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name/noredirect/true">Page without redirect</a></td>
<td>Turning off redirection, if the <a href="http://www.wikidot.com/doc:redirect-module">Redirect Module</a> is present on the page</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name/tags/tag1,tag2 Set tags]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name/tags/tag1,tag2">Set tags</a></td>
<td>Sets tags on the page via URL, comma-delimited</td>
</tr>
<tr>
<td><tt><span style="white-space: pre-wrap;">[http://site-name.wikidot.com/page-name/norender/true No Render]</span></tt></td>
<td><a href="http://site-name.wikidot.com/page-name/norender/true">No Render</a></td>
<td>Goes to the page, but does not render it. It allows to change the source of the page when there is a problem with page functionality</td>
</tr>
</table>
