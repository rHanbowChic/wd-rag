
<div class="page-title">
                    The 'date' field type
                </div>
<div id="page-content">
<div class="alert alert-info">
<p>October 2, 2014<br/>
This is a work in progress. Some options still need to be tested before being added to this documentation.</p>
</div>
<br/>
Defines a date input field that uses the <a href="http://api.jqueryui.com/datepicker/" target="_blank">jQuery UI Datepicker Widget</a> to select dates. It uses most of the options available that are of type <em>String</em> or <em>Int</em>. A basic datepicker can be added using just the <em>type</em> property:
<div class="code">
<pre>
<code>[[form]]
fields:
  date-1:
    type: date
[[/form]]</code>
</pre></div>
<p>The specific properties you can use on a text field:</p>
<ul>
<li><strong>width</strong>: specifies the visible field width in columns (fixed spaced characters, more or less). If the <em>autoSize</em> option is enabled, it overrides this property setting.</li>
<li><strong>options</strong>: specifies the jQuery UI Datepicker Widget options to apply.</li>
</ul>
<p>Here is an example using a few of the available options:</p>
<div class="code">
<pre>
<code>[[form]]
fields:
  mydate:
    type: date
    label: 'My Date Widget'
    options:
      appendText: ' This Demo is Cool!'
      autoSize: true
      changeYear: true
      dateFormat: 'DD, d MM yy'
      firstDay: 1
      showOn: button
      yearRange: '2014:2025'
[[/form]]</code>
</pre></div>
<br/>
The above example creates a datepicker calendar widget with the following options:
<ul>
<li><strong>showon: button</strong> adds a button after the input field and it must be clicked to open the datepicker.</li>
<li><strong>autoSize: true</strong> automatically sets the width of the input box to fit the date format defined in the <em>dateFormat</em> option.</li>
<li><strong>changeYear: true</strong> creates a dropdown selector of changing the year. This is useful if you don't want to make users click the previous/next links 12 times to move to another year.</li>
<li><strong>dateFormat: 'DD, d MM yy</strong> formats the date selected to something like "Wednesday, 1 October 2014". See the date format reference below for more details on date formats.</li>
<li><strong>firstDay: 1</strong> tells the calendar to use Monday as the first day of the week for the widget's calendar instead of the default of Sunday (0=Sunday, 1=Monday, …, 6=Saturday).</li>
<li><strong>yearRange: '2014:2025'</strong> limits the year selection to the range specified.</li>
</ul>
<p>Dates in a date field are stored as a date number and displayed based on the <em>dateFormat</em> option you specifiy. add the <em>altFormat</em> option to use a different date format for your alternate date. If you want to save and use the date as <em>text</em> and not a number, you can use the <em>altField</em> option to place a text version of the date into another field in your data form. The example below will place a copy of the date selected in the <strong>mydate</strong> field as text into the <strong>alt-date</strong> field text box on the form. <strong><span style="white-space: pre-wrap;">%%form_data{alt-date}%%</span></strong> will be stored in the data form as text.</p>
<h2 id="toc0"><span>altDate/altFormat Example</span></h2>
<div class="code">
<pre>
<code>[[form]]
fields:
  mydate:
    type: date
    label: 'This date will fill in'
    options:
      appendText: ' altField Demo'
      autoSize: true
      changeYear: true
      dateFormat: 'DD, d MM yy'
      altField: 'input[name=field-alt-date]'
      altFormat: 'm/d/yy'
      yearRange: '2014:2025'
  alt-date:
    type: text
    label: 'Filled in by date above'
    width: 10
[[/form]]</code>
</pre></div>

<h2 id="toc2"><span>Datepicker Widget Options:</span></h2>

<td><strong>altField</strong></td>
<td>altField: 'input[name=field-&lt;target field name&gt;]'</td>
<td>The alternate field on your form to place a text copy of the date based on the current <em>dateFormat</em> option. Use <em>altFormat</em> to define a different date format.</td>
<td>altField: 'input[name=field-alt-date]'</td>
</tr>
<tr>
<td><strong>altFormat</strong></td>
<td>altFormat : format string</td>
<td>Used to apply an alternate <em>dateFormat</em> to the <em>altDate</em> option</td>
<td>altFormat: 'm/d/yy'</td>
</tr>
<tr>
<td><strong>appendText</strong></td>
<td>appendText : 'string to display'</td>
<td>The text to display after each date field.</td>
<td>appendText: ' This Demo is Cool!'</td>
</tr>
<tr>
<td><strong>autoSize</strong></td>
<td>autoSize : true|false</td>
<td>Set to true to automatically resize the input field to accommodate dates in the current dateFormat.</td>
<td>autoSize: true</td>
</tr>
<tr>
<td><strong>buttonImage</strong></td>
<td>buttonImage : 'url of image file'</td>
<td>URL of an image to use to display the datepicker when the showOn option is set to "button" or "both".</td>

</tr>
<tr>
<td><strong>buttonImageOnly</strong></td>
<td>buttonImageOnly : true|false</td>

<td>buttonImageOnly: false</td>
</tr>
<tr>
<td><strong>buttonText</strong></td>
<td>buttonText : 'string to display'</td>
<td>The text to display on the trigger button. Use in conjunction with the showOn option set to "button" or "both". If <em>buttonImage</em> is set, the text becomes the alt value and is not directly displayed.</td>
<td>buttonText: 'Pick!'</td>
</tr>
<tr>
<td><strong>changeMonth</strong></td>

<td>changeYear: true</td>
</tr>
<tr>
<td><strong>closeText</strong></td>
<td>closeText : 'string to display'</td>

<td>closeText: 'Abort Mission'</td>
</tr>
<tr>
<td><strong>currentText</strong></td>
<td>currentText : 'string to display'</td>
<td>The text to display for the close link. Use <em>showButtonPanel: true</em> to display this button.</td>
<td>currentText: 'Go to Today'</td>
</tr>
<tr>
<td><strong>dateFormat</strong></td>
<td>dateFormat : format string</td>

<td>dateFormat: 'DD, MM yy'</td>
</tr>
<tr>
<td><strong>dayNames</strong></td>
<td>dayNames : [array of names]</td>
<td>The list of long day names, starting from Sunday. Useful for languages other than English. Used with the <em>DD</em> date format option.</td>
<td>dayNames: [Sonntag, Montag, Dienstag, Mittwoch, Donnerstag, Freitag, Samstag]</td>
</tr>
<tr>
<td><strong>dayNamesMin</strong></td>
<td>dayNamesMin : [array of names]</td>
<td>The list of minimised day names, starting from Sunday, for use as column headers within the datepicker. Useful for languages other than English.</td>
<td>dayNamesMin: [So, Mo, Di, Mi, Do, Fr, Sa]</td>
</tr>
<tr>
<td><strong>dayNamesShort</strong></td>
<td>dayNamesShort : [array of names]</td>
<td>The list of abbreviated day names, starting from Sunday. Useful for languages other than English. Used with the <em>D</em> date format option.</td>
<td>dayNamesShort: [Son, Mon, Die, Mit, Don, Fre, Sam]</td>
</tr>
<tr>
<td><strong>defaultDate</strong></td>
<td>defaultDate : 'date string'|+/- number of days from today|string of values and periods</td>
<td>Set the default date on first opening of the widget. Specify either an actual date via a string in the current dateFormat, or a number of days from today (e.g. +7) or a string of values and periods ('y' for years, 'm' for months, 'w' for weeks, 'd' for days, e.g. '+1m +7d'), or null for today.</td>
<td>defaultDate: '+1m -1d'</td>
</tr>
<tr>
<td><strong>duration</strong></td>
<td>duration : number of milliseconds|slow|normal|fast</td>

<td>duration : slow</td>
</tr>
<tr>
<td><strong>firstDay</strong></td>

<td>firstDay: 1</td>
</tr>
<tr>
